Elementos sem inputs

até agora todos os elementos utilizados foram do tipo input, e são vários deles, mas também existem elementos que não são parte do input, segue abaixo um exemplo prático deles

select

select é aquela caixinha com dropdown geralmente usada para selecionar seu estado mas também pode ser usada em várias outras coisas já que ela é altamente personalizável

uma coisa legal que podemos fazer é agrupar as opções do select utilizando o optgroup, dessa forma vai ser uma lista mais didática e organizada

para escolher o nome do optgroup precisamos adicionar o elemento label a ele

também é possivel selecionar uma opção como o ponto inicial, adicionando o element selected a opção desejada caso contrário a opção marcada vai ser a primeira da lista

Datalist

select é bem útil pra selecionar uma opção de uma lista pré-definida, mas e se você também quiser que, além da lista, o usuário tenha liberdade para escrever sua prórpia opção?

para isso vamos utilizar o datalist

apesar da datalist não ser um tipo de input, ela precisa estar conectada a um, nesse caso, existe um input do tipo text conectado a uma datalist

textarea

as vezes o usuário precisa de um espaço grande para dar feedback ou até mesmo fazer um pedido, o que pode ser feito com textarea, isso vai criar uma região bem grande onde você pode selecionar o tamanho vertical e horizontal


é uma tag bem intuitiva, lembre-se de utilizar o post para essa, pois o get tem limite de 3000 bytes
